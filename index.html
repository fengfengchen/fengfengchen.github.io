过渡效果

Vue 在插入、更新或者移除 DOM 时，提供多种不同方式的应用过渡效果。
包括以下工具：

在 CSS 过渡和动画中自动应用 class
可以配合使用第三方 CSS 动画库，如 Animate.css
在过渡钩子函数中使用 JavaScript 直接操作 DOM
可以配合使用第三方 JavaScript 动画库，如 Velocity.js

进入/离开

这种情况下一般是针对单元素或者是组件来说的，添加效果的方式很简单，只要在单元素或者组件外面套一层标签即可.

当插入或删除包含在 transition 组件中的元素时，Vue 将会做以下处理：

  1. 自动嗅探目标元素是否应用了 CSS 过渡或动画，如果是，在恰当的时机添加/删除 CSS 类名。

  2. 如果过渡组件提供了 JavaScript 钩子函数，这些钩子函数将在恰当的时机被调用。

  3. 如果没有找到 JavaScript 钩子并且也没有检测到 CSS 过渡/动画，DOM 操作 (插入/删除) 在下一帧中立即执行。(注意：此指浏览器逐帧动画机制，和 Vue 的 nextTick 概念不同)

Vue从入门到天意之组件进阶

发表于 2018-10-25

组件进阶

组件的名字

推荐使用 kebab-case(短横线分隔命名）的方式来命名一个组件，引入的时候，可直接使用

prop属性

组件基础提到过prop属性通过一些自定义一些html属性去获得父组件的数据，这是一个单向的数据流，即数据只能从父组件流向子组件。反过来则不行

实际上prop属性值可以是任何类型，常见的是数组和对象。

当需要对prop属性值指定类型的时候，可以对其进行验证，可以利用原生构造函数，也可以自定义一个函数

事件

事件名应始终使用 kebab-case 的形式。

插槽

Vue 实现了一套内容分发的 API，这套 API 基于当前的 Web Components 规范草案，将 元素作为承载分发内容的出口。

具名插槽：
有些时候我们需要特定的结构来展现内容，这个时候就需要给添加一个name属性,并为其设置一个值。那么我们就可以在组件实例上使用slot属性，向特定的结构插入内容,而没有slot属性的内容将填充在没有name属性的上

  1. jq中的index()方法理解：
    $(this).index() 可以获取当前元素在同级父级元素下的索引位置，若此对象存在拥有不同父级的情况，那么他们的索引值不会一直叠加；需要用到下面的这种形式
    $(father obj).index(thisobj),这样的形式可以获取当然的对象在其集合对象的索引位置，即使这些对象不再同一级父亲下。
  2. css元素选择器：
    兄弟选择器 +：若为h2 + p 形式则表示只选择h2紧接着的p选择，此时只有一个元素被选择，若为 p + p 形式，则表示选择拥有相同的父元素的除了第一个p元素的其他所有p;
