Vue学习记录

发表于 2018-07-01

html部分:

内容中 输出内容；
v-？ 表示Vue指令，放在标签头中，用于各种指令
v-bind:/: 绑定一个标签属性，属性值需用引号包裹，可以是一个变量、数组。字符串拼接变量等
v-once 绑定一个变量，只变化一次
v-text 输出变量内容作为标签内容
v-html 区别在于可以转化HTML格式
v-model 双向绑定变量，变量值改变，会反应在标签中，标签内容改变，也会改变变量值
v-for 循环指令
v-if/v-else v-if/v-else-if…/v-else 条件判断
v-show 控制显示隐藏 效果比v-if要好点

script部分

var app = new Vue({}); 实例化Vue对象，内容用属性的键值对表示
el 表示一个实例化对象，值一般是#ID，字符串
data 存放变量数据的地方，本身是一个对象，对象中的元素值可以是字符串，对象，数组之类的
computed 计算属性

组件

props: 子组件的属性变量存放，通常会将父组件的值作为属性值传给子组件
slot 内容分发，在模板中定义slot标签的位置，在渲染过程中会将实际标签将其替换，有一个name属性，可以定义实际渲染的部分
scrop: 将子组件传递过来的props对象传递给一个临时变量，然后可以读取该临时变量的一些值

开启个人博客

发表于 2018-05-30

一直都没有写东西的习惯，一是因为文学功底不够，另一个是比较排斥。但是呢，好记性不如烂笔头，基于此，还是决定克服自身的这种排斥情绪。

这第一篇文章就记录一下这个博客的从无到有好了。

jekyll VS hexo

一开始是准备使用jekyll的，知名度还挺高的，就是对于windows系统不大友好，我也没有找到好的中文教程，官方的教程我照做了，就是没成功。。。所以我最后选择了hexo，只要有npm & node 就行，对于一个做前端的来说，完全是标配啊。

过程

  1. 打开命令行工具（cmd），并在全局环境下安装hexo工具；

  2. 在某个目录下新建blog网站的文件夹，并初始化；

  3. 进入这个项目，并安装相应的包文件；

  4. 本地预览，开启本地服务器；

到此，基本上你就可以预览到官方的默认hello页面了，当然这只是本地的，别人是访问不到的，下一步，我们需要将blog部署到线上，让别人也能看得到；

部署到线上

你可以单独弄一个服务器或者也可以使用别人提供的这种服务，比如说 GitHub 和 码云 都是提供这种服务的，就是那个pages服务，我选择的是码云，那就以码云为例好了。
首先：需要在码云上建立一个仓库，这个从仓库的名字最好是你的英文名字，或者一个看起来好不错的英文，
仓库建好之后，复制仓库的url地址，打开本地项目下一级目录中的_config.yml文件（这个是配置文件），找到deploy字段（最底部），替换ropo的值为刚才复制的地址。
其次：安装自动部署工具

然后： 使用命令部署发布

第一部署发布需要验证码云的账号密码，稍等一下，提示发布成功之后，访问你建立的那个项目中的pages服务，你会看到一个类似于这个https://canbelieve.gitee.io的地址，访问你的这个地址，就能看到你发布的内容啦。

到此，一个简单的blog就建立成功了，之后还有主题，导航，评论等内容，就需要自己去寻找思路了。
